## What landed this session

### Track 1 speedup sprints (A/B sim wall-time)

- **1A — Parallel pods.** `compare()` dispatches multi-pod runs through
  `ThreadPoolExecutor` (workers = `min(len(pods), cpu_count())`). Each
  pod has its own Forge subprocess; GIL released during the wait.
  `parallel=False` flag preserves sequential behavior for tests.
  Aggregation happens as pods complete; final `report.pods` is sorted
  back to original input order.
- **1B — Adaptive early-stop.** New `_is_decisive(old_wins, new_wins,
  games_remaining)` returns True iff `|margin| > games_remaining`. When
  it fires, queued pod futures get canceled (parallel) or the loop
  breaks (sequential). Guarded so it never fires after a single pod.
  `early_stop=True` by default; new `report.stopped_early` field +
  `pods_planned` reflect what actually ran.
- **1C — Per-pod intra-pod abort.** `forge_runner._run_streaming` gained
  `abort_check(line) -> bool` callback. Per-line callback parses
  `Game Result: …` and kills the JVM as soon as the in-pod margin
  exceeds games remaining. `_synthesize_match_result(state)` builds a
  `Match Result:` line so log_parser still sees a complete record.
  Pod result dict gains `intra_pod_aborted: bool` and
  `games_actually_played: int`.
- **1E — Auto-scaled filler_pairs.** New `auto_filler_pairs()` returns
  `min(4, max(2, cpu_count() or 2))`. Web app uses it instead of
  hardcoded `2`. Multi-core boxes get tighter verdicts at same
  wall-time.
- **1F — UI surfacing.** `/api/propose_swap` response now includes
  `pods_completed`, `pods_planned`, `stopped_early`, `pod_summaries`.
  Result panel renders "Pods 2/4 (stopped early — verdict locked)"
  and "Intra-pod stop: Pod 2 stopped at 3/5".
- ETA copy in `app.js` updated: pod 5g estimate now 75s (was 180s
  sequential).

  Close button stays reachable on tall content (the prior issue: user

- **Soft refresh.** New `selectDeck(deckId, li, { soft: true })` keeps

  lookups for new cards.
- **Sub-100 source padding.** `_pad_main_to_99()` tops up short decks
  (e.g. Goblin at 71 main) with basic lands matching the deck's
  existing color distribution. UI shows "Source was 71/100; padded
  with 28 Mountain". Audit response includes `basics_padded` and
  `basics_padded_breakdown`.

  `vendor/_js_errors.log`. Returns a ref token (`20260429023316-a1b2`)

  surface here automatically.
- **Bracket auto-inference.** Dashboard emits `inferred_bracket`
  alongside the declared `bracket`. UI warns when heuristic suggests
  higher than declared ("Heuristic suggests B4 — more powerful than
  declared").
- **Stale-file cleanup at boot.** `_cleanup_stale_staged_files()`
  sweeps interrupted `*_proposed_<ts>.dck` / `*_converted_<ts>.dck`
  files older than 60s. Runs in `create_app`.
- **Moxfield format fix.** `to_moxfield_line()` in `moxfield_push.py`

  `1 Arcane Signet (MIC) 157`. `dck_to_textarea` runs every line

### Track 2 prep (forge_py multi-deck sim)

- **New module `forge_py_correlation.py`.** Runs `forge_py.combat.
  run_multiplayer_game` alongside Forge for paired-verdict logging.
  `run_forge_py_ab(old, new, games, mode)` returns
  `ForgePyABResult`. `log_correlation_row(...)` appends CSV.
  `correlation_summary(log_path)` reports `{rows, agree, disagree,
  agreement_rate, errors}`.
- **CLI**: `python -m commander_builder.forge_py_correlation` prints
  summary; `--json` for machine-readable.
- **Endpoint**: `/api/correlation_summary` returns the summary +
  whether the harness is currently enabled.
- **Topbar surface**: `loadHealth()` pulls the summary; badge shows
  "forge_py 75% agree (12)" when rows exist.
- **Opt-in**: `COMMANDER_BUILDER_CORRELATE_FORGE_PY=1` env var. OFF
  by default — adds 5-15s per propose-swap.
- Imports `forge_py` lazily; missing install never breaks the main
  flow (returns `ForgePyABResult(error="forge_py not importable")`).

## Decisions explicitly made (with rationale)

- **Sprint 1C was reframed** from "JVM persistence" to "per-pod
  intra-pod abort." Original required Forge source modifications
  (its `sim` mode runs once and exits) and only saved ~10s per pod.
  Reframed sprint required zero Forge changes and saves 20-50% on
  lopsided matches. See `docs/SPEEDUP_TRACK_1.md`.
- **Sprint 1D (result cache) was skipped.** Real-use cache hits are
  rare (each propose-swap stages a fresh proposed file). Spec doc
  flagged it as "0.5 day, opportunistic." Skip stands until someone
  has a use case.
- **Track 2 harness runs forge_py opportunistically, not as a fast
  path.** forge_py's combat engine is shallow (single-attacker, no
  flying/reach/first-strike); 90%-accurate forge_py that
  systematically misjudges control vs aggro is worse than waiting
  3 min for Forge ground truth. Plan: accumulate ≥30 paired rows,
  compute Pearson r per archetype, only flip default when r ≥ 0.90
  per archetype.
- **forge_py is NOT a hard dependency.** Imported lazily inside the
  correlation module so a missing install never breaks
  commander_builder. forge_py lives at `C:/dev/forge_py/` and is
  importable today (verified via `python -c "import forge_py"`).
- **All MTG repos are public on GitHub.** Reinforced as a memory rule
  (`feedback_public_repo_safety.md`). Scan diffs for `sk-`, `Bearer `,
  JWT prefixes, `.env` content, personal emails before commit. Test
  fixtures use placeholder keys like `"sk-test-byo-12345"`.
