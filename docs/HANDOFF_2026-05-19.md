## Setup checklist

### 1. Clone the repo

```bash
git clone https://github.com/LlamaAdam/commander-builder.git
cd commander-builder
git checkout feature/2026-04-28-session
```

### 2. Install Python deps

Requires **Python 3.10 or newer** (pinned in `pyproject.toml`).

```bash
# Editable install so the commander-* CLIs are on PATH after a pull.
pip install -e .[claude]
```

The `[claude]` extra pulls `anthropic>=0.40`. Without it, the auto-curate CLI degrades gracefully to the heuristic-only path.

### 3. Set up credentials (ONE-time, outside the repo)

The auto-curate pipeline needs `ANTHROPIC_API_KEY`. It lives at `~/.commander-builder/credentials` so it can never accidentally get committed to this public repo.

```bash
commander-config init
# Open the file the command printed and paste your key after
# the commented `ANTHROPIC_API_KEY=` line.

# Unix only ‚Äî restrict file mode to your user:
chmod 600 ~/.commander-builder/credentials
```

Full details + edge cases in [docs/SECRETS.md](SECRETS.md). You can skip this step entirely if you only intend to run the heuristic advisor and never need a Claude curator call.

### 4. (Optional) Vendor Forge for A/B sims

If you want `commander-auto-curate --run-sim` and the A/B sim CLI commands to work, install Forge under `vendor/`:

```
vendor/
‚îú‚îÄ‚îÄ README.md                        # committed; tells you the expected layout
‚îú‚îÄ‚îÄ jre/                             # OPTIONAL portable JRE (bundled Java 17+)
‚îÇ   ‚îî‚îÄ‚îÄ bin/{java.exe,java}
‚îú‚îÄ‚îÄ forge/                           # Forge 1.6.62+ install
‚îÇ   ‚îî‚îÄ‚îÄ forge-gui-desktop-X.Y.Z/
‚îÇ       ‚îú‚îÄ‚îÄ forge-gui-desktop-X.Y.Z-jar-with-dependencies.jar
‚îÇ       ‚îú‚îÄ‚îÄ res/                     # required ‚Äî card data lives here
‚îÇ       ‚îî‚îÄ‚îÄ ...
‚îî‚îÄ‚îÄ knowledge_log.sqlite             # auto-created on first iteration log
```

Both `vendor/jre/` and `vendor/forge/` are `.gitignore`d. The verifier (`commander-doctor`) checks the bundled JRE first, then falls back to the system PATH `java`. Skip the vendor JRE if you have system Java 17+ installed.

You don't need Forge for the web dashboard, the advisor, the auto-curator's apply step, or the test suite ‚Äî only for the Forge A/B sim verdict loop.

### 5. (Optional) Set up the shared card cache

Scryfall card data, oracle text, and now images are cached under a shared `mtg_cards/` directory. The lookup order:

1. `MTG_CARDS_DIR` environment variable (override / CI).
2. `C:\dev\mtg_cards` if it exists (the canonical local path on the original dev box).
3. `<repo>/.cache/` as a fallback.

On a fresh machine, the project will land at #3 and silently build out the cache on first audit. If you want to share the cache with another `commander-builder` clone (or with the sister `forge_py` project), set `MTG_CARDS_DIR` to a common path:

```bash
# bash / zsh
export MTG_CARDS_DIR="$HOME/mtg_cards"

# Windows PowerShell
$env:MTG_CARDS_DIR = "C:\dev\mtg_cards"
```

## Common workflows

```bash
# Import a Moxfield deck as your baseline
commander-import --user https://moxfield.com/decks/<id>

# End-to-end auto-curate: advisor ‚Üí Claude curator ‚Üí apply ‚Üí optional sim
commander-auto-curate "[USER] My Deck [B3].dck" --bracket 3 --run-sim

# Modes: polish (default, 5+5 swaps), overhaul (15+15), free (unbounded)
commander-auto-curate "[USER] My Deck [B3].dck" --bracket 3 --mode overhaul

# Heuristic-only advice (no API spend, no Claude required)
commander-advise --user "[USER] My Deck [B3].dck" --bracket 3

# Refresh the hardcoded card lists in deck_health against current Scryfall
python scripts/refresh_card_lists.py --only mdfc

# Web dashboard
python -m commander_builder.web --port 5050
```

Full CLI catalogue in [README.md](../README.md#cli-commands).

## What you can't get from git (machine-specific data)

These files / dirs are `.gitignore`d and must be reconstructed or copied manually:

| Path | Purpose | How to recreate |
|---|---|---|
| `~/.commander-builder/credentials` | `ANTHROPIC_API_KEY` | `commander-config init` + paste your key |
| `vendor/forge/` | Forge install (~200 MB) | Download Forge 1.6.62+ from SourceForge (`https://sourceforge.net/projects/cardforge/files/`) or the project's GitHub releases; extract under `vendor/forge/` per `vendor/README.md` |
| `vendor/jre/` | Portable Java 17+ runtime | Optional ‚Äî only if you don't want system Java on PATH |
| `vendor/knowledge_log.sqlite` | Iteration history | Auto-created on first iteration log; copy from old machine to preserve verdict history |
| `mtg_cards/` (or `.cache/`) | Scryfall JSON + image cache | Auto-built on first audit; copy from old machine to skip the cold-cache rebuild (~5-10 min over Scryfall's rate limit) |
| `vendor/forge/userdata/decks/commander/*.dck` | Your deck library | Copy from old machine ‚Äî these are your input data |

If you want to seed the iteration log to test the dashboard immediately:

```bash
python scripts/seed_demo_knowledge_log.py
```

That creates a few synthetic iterations for a fake "Omnath" deck so the iteration history / verdict-breakdown / pricing-sparkline panels populate without you needing real data.

## Recent feature highlights (2026-05-15 ‚Üí 2026-05-19)

Full breakdown in [CHANGELOG.md](../CHANGELOG.md). The highlights you'd care about if you only knew the project from before 2026-05-15:

- **`commander-auto-curate` CLI** ‚Äî end-to-end pipeline. `--run-sim` flag closes the loop via Forge A/B with empirical kept/reverted verdicts logged to knowledge_log.
- **Defensive filter chain** in the curator: WotC bracket caps (3-card B3/B4 game-changer limit), Scryfall color identity, protected-card list, balance-and-pad to legal 99-main.
- **5-tile deck-health row** in the audit panel (MDFCs, spell density, mana sinks including activated-ability detection, wincon protection, self-mill).
- **Manual verdict UI** ‚Äî `PATCH /api/iterations/<id>/verdict` plus Kept/Reverted/Neutral buttons next to pending iterations in the dashboard graph.
- **Local card-image cache** at `/api/card_image/<size>/<name>` ‚Äî first view of any deck still pays the Scryfall cold-fetch cost, but every subsequent view serves from local disk. Killed the 30-60s Chrome stall on big advisor outputs.
- **Tribal-deck heuristics** ‚Äî Three Tree City baseline, Nykthos auto-include for mono-color tribal decks (devotion-scaling kicks in), Arcane Signet now classifies as ramp via a natural-language regex.
- **Fast/slow pytest lanes** ‚Äî `pytest` is now 30-40s by default. Use `pytest --run-slow` for the full 1233-test suite.

## Quick "did the move work?" sanity script

After copying your machine-specific data (credentials, vendor/forge, deck library, optional knowledge_log + mtg_cards), run:

```bash
commander-doctor                                         # all GREEN?
pytest                                                   # 1110 passed, 33s?
python -m commander_builder.web --port 5050 &
curl -s http://127.0.0.1:5050/api/health | python -m json.tool
# {"status": "ok", "deck_dir": "...", "deck_count": N}
```

If `commander-doctor` flags RED, fix that first; the web app won't behave well with a broken Forge or stale credentials.

If `pytest` reports skip counts other than ~123, the `--run-slow` opt-in marker contract may have drifted ‚Äî re-read `tests/conftest.py` to see what's going on.
